Blinkit Real-Time Demand Analytics
Business Problem

Quick-commerce platforms like Blinkit must understand order demand patterns and peak ordering times to optimize delivery operations and inventory planning.

This project analyzes hourly demand patterns using a modern data analytics pipeline built with Python, SQL, Google BigQuery, and Tableau.

The goal is to identify peak demand hours and overall order trends to support better operational decision-making.

Project Overview

This project builds an end-to-end analytics pipeline starting from raw datasets and ending with an interactive dashboard.

The workflow includes:

Data ingestion from CSV files
Data cleaning and transformation using Python
Loading processed data into Google BigQuery
Running analytical SQL queries
Creating an interactive Tableau dashboard

This simulates a real-world data analytics workflow used in modern data teams.

Objective
Analyze hourly demand trends
Identify peak demand hours
Measure total order volume
Visualize insights using a dashboard
